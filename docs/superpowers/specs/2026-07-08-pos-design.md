# 手機配件店 POS 系統設計

2026-07-08 定案。單機使用,手機配件店(上千件商品),搭配條碼掃描器。

## 1. 整體架構

```
單一 .exe(PyInstaller onefile)
  └─ 啟動 → 內建 localhost 伺服器(FastAPI)
       ├─ 後端 API(Python)
       ├─ 前端靜態檔(HTML/CSS/JS + Vue 無建置模式)
       └─ 自動開瀏覽器 → http://127.0.0.1:port
資料落地(集中 data/ 一個資料夾):
  data/ ├─ pos.db(SQLite)
        ├─ backups/(自動備份,GFS 輪替)
        └─ config.json(設定:埠號、條碼機等)
```

- **前端:Vue 無建置模式**——`vue.js`(production 版)下載一份放專案內、包進 exe,離線可用;**不引入 npm/node/vite 建置鏈**(維護者「檔案不灑亂」需求)。前端依畫面拆多個 `.js` 檔,模板寫在 HTML `<template>` 標籤(保留語法高亮)。日後若規模成長到無建置模式難以維護,前端可單獨遷移 Vite 專案,API 層隔離、後端不動
- **後端:FastAPI**;**DB:SQLite**(單機零安裝、單檔備份、完整 transaction)。未來多機:後端資料層換 PostgreSQL 即可,前端不動
- 條碼掃描器＝鍵盤輸入,前端聚焦輸入框接字串,免驅動
- 條碼列印:後端預留「列印服務」介面,先不接實體機,之後接特定條碼機只實作這層

## 2. 資料結構

### 商品

```
Product(款)
  ├─ product_id, 名稱, 分類, 預設售價(可空), 備註
  └─ Variant(變體/SKU)
       ├─ variant_id, product_id
       ├─ attributes(彈性屬性 JSON,如 {規格:霧面, 手機型號:iPhone17pro})
       └─ 售價(可空,覆蓋款預設價)

Barcode(條碼,獨立表,多對一)
  ├─ barcode(唯一), variant_id, 來源(原廠/店內自產)
  └─ 一個變體可掛多組條碼(混合模式:原廠 EAN＋店內自產並存)
```

- 店內自產條碼加前綴區隔,避免與 EAN 撞號
- 既有 `TL100000001` 式編號＝店內條碼,匯入時掛進 Barcode 表,掃了即可結帳

### 屬性欄位與選單庫(預設下拉)

沿用維護者 Excel(`產品清單_org.xlsm`)的建檔體驗:

```
AttributeField(屬性欄位定義)
  ├─ 名稱、顯示順序、是否啟用
  └─ 標題可改(「分類1」→「顏色」),可新增欄位,全程式跟著換

AttributeOption(選項庫)
  ├─ 屬性欄位, 選項值, 排序, 是否啟用
  └─ parent 條件(選填)→ 連動下拉
       例:「規格=亮面」parent=「廠牌=HODA」
       例:「型號=iPhone17pro」parent=「手機品牌=iPhone」
```

- 系統預設建好八欄:商品種類/廠牌/規格/商品描述/分類1/分類2/手機品牌/手機型號
- 建檔下拉依 parent 連動過濾;**可下拉也可手打**,手打新值問「加入選單?」→ 選單越用越齊
- 設定頁可整理選單(改名/排序/停用)

### 庫存(異動流水制,不存死數字)

```
StockMovement(庫存異動)
  ├─ variant_id, 數量(進貨+/銷售-/盤點±), 類型, 時間, 來源單號, 備註
  └─ 某變體庫存 = SUM(該變體所有異動)
```

- 庫存允許負數(先賣後補的現實),畫面標紅提醒

### 銷售

```
Sale(交易)                 SaleItem(明細行)
  ├─ sale_id, 時間            ├─ sale_id, variant_id
  ├─ 付款方式(清單可設定)     ├─ 數量, 成交單價(記當下實際價)
  ├─ 整單折抵/折扣            └─ 單品折扣
  └─ 應收/實收/找零
```

- 建檔可不填價;收銀時可臨櫃改價,成交價一定記在 SaleItem
- 不做多組價、不做價格歷史

### 盤點

```
StocktakeSession(盤點單)
  ├─ session_id, 起迄時間, 操作人, 狀態(進行中/已結案), 備註
  └─ StocktakeItem(明細)
       ├─ variant_id
       ├─ system_qty(開盤當下系統量快照——盤點中可能仍有銷售)
       └─ counted_qty(實盤量;差異＝算出,不存)
```

- **局部盤**:只調有盤到的變體,支援分區分次盤,沒碰的不動
- 結案:對有差異的變體各寫一筆 ADJUST 異動(±,來源單號=session_id),可回查

## 3. 作業流程

### 收銀(主畫面)
1. 游標常駐掃描框 → 掃條碼 → 查 Barcode → 帶出變體+價格入購物車
2. 無價商品 → 跳輸入框要求填成交價
3. 可改單價、單品折扣、整單折抵
4. 選付款方式 → 結帳 → 同一 transaction 寫 Sale＋SaleItem＋每品項 StockMovement(−)
5. 查無條碼 → 提示,可當場快速建檔或略過

### 進貨(點貨入庫)
1. 掃條碼 → 已存在:輸數量,寫 StockMovement(+)
2. 不存在 → 快速建檔(連動下拉,價格可空)
3. 無原廠條碼散件 → 系統產店內條碼掛上 → 預留送條碼列印

### 盤點
1. 開盤點單 → 逐件掃(重複掃=累加)或手輸
2. 可中斷續盤(進度存盤點單)
3. 結案:差異清單 → 確認 → 寫 ADJUST 異動

### 銷售紀錄
- 依日期/付款方式/商品篩選、看單筆明細、日結統計(各付款方式小計)、CSV 匯出

## 4. 錯誤處理與測試

- 結帳寫入包同一 transaction,不會「有銷售沒扣庫存」
- 啟動自動備份至 `data/backups/`(GFS 輪替)
- 純邏輯(條碼解析/庫存加總/折扣計算/盤點差異/連動選單過濾)進 `tests/` 單元測試

## 5. 初始資料匯入

- 從 `產品清單_org.xlsm` 匯入:商品資料庫 575 筆(含 `TL...` 編號當店內條碼)＋整個選單庫(含連動關係)
- 一次性匯入腳本,不入常駐程式

## 6. 不做的事(YAGNI)

- 多機連線(架構天然可擴充,先不做)
- 會員/點數
- 多組價格、價格歷史
- 退換貨(之後要再議)
- 發票機/收據列印(之後要再議)
